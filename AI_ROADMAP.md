# 세방 안전운임 웹/앱 — AI 확장 로드맵

## 비전
정적 **조회 도구** → 대화형 **물류 어시스턴트** → **AI 기반 최적 배차 플랫폼**

이미 앱이 보유한 자산(운임 데이터 · 부대조항 전문 · 전국 행정동 데이터 · 할증/복화/COMBINE 규칙)을
연료로, 앱을 갈아엎지 않고 모듈처럼 단계적으로 확장한다.

## 관통 원칙 (정확성·안전성)
> **AI는 "추출·검색", 사람은 "확인", 금액·오더·배차 확정은 "검증된 로직"이 처리한다.**
- AI가 운임 금액을 추론하지 않는다(고시에 법적으로 묶인 정확값 → 환각 금지).
- AI 답변·추출 결과는 항상 근거(조항/문서 원문)를 함께 제시하고, 모르면 모른다고 답한다.
- 사람 확인 단계를 거친 뒤에만 오더/배차로 확정된다.

---

## 단계별 백로그

### Phase 1 — 음성 입력 (모바일 전용)  ·  난이도: 하  ·  ☑ 1차 구현 완료(검증 중)
- [x] 모바일·터치 기기에서만 음성검색 버튼 노출 (PC는 숨김) — v185~v190
- 구현: 🔊 버튼 → 음성인식(ko-KR) → 최상위 자동선택 → 결과 음성안내(TTS)
       · 기본값 왕복+안전위탁 · 안드로이드 정상 · iOS는 실기기 검증 중(인식 실패 시 키보드 받아쓰기 폴백)
- 모바일 브라우저 음성인식(Web Speech API)으로 **기존 검색창에 텍스트만 입력**
- AI 역할: 음성→텍스트(STT)뿐 → **추론 없음 = 정확성 문제 없음**
- 의존성: 없음(순수 클라이언트, 백엔드 0) → 가장 빠른 실물화
- 대상 가치: 차주/현장 사용성

### Phase 2 — 부대조항 AI 질의응답 (RAG)  ·  난이도: 중
- [ ] "복화운송이면 운임 어떻게 돼?" 등 자연어 질문 → 쉬운 말로 답변
- **항상 근거 조항("제○조 (제목)")을 인용**, 조항에 없으면 "고시에 명시되지 않음"으로 솔직히 답함
- 데이터: 이미 보유(`window._CL` 34개 조항 전문) → 추가 수집 불필요
- 의존성: Claude API 1개(소형 백엔드/프록시)
- 대상 가치: 복잡한 고시 해석 부담 ↓ (화주·차주·영업 공통)

### Phase 3 — B/L 접수 → 사내 오더생성 연계  ·  난이도: 중상
- [ ] 문서(사진/PDF) 업로드 → AI 추출 → 사람 확인 → 세방 오더생성 프로그램 전송
- 추출 필드: 목적지 · 컨테이너 규격/수량 · 중량 · 품목(할증 판단) · 컨/부킹번호 등
- 주의: B/L엔 보통 "항구"까지만 있음. **내륙 최종 배송지·품목 할증 조건**은
  운송의뢰서/작업지시서에 있는 경우가 많음 → 추출 대상 문서를 그에 맞게 선택
- 의존성: 사내 오더 프로그램의 **API/연동 창구**(세방 IT 협조)
- 대상 가치: 수기 오더 입력 자동화 → 시간·오타 대폭 절감

### Phase 3.5 — 협력사 거래명세서 자동 대사(운임 검증)  ·  난이도: 중상
협력사가 보낸 거래명세서의 청구 금액이 **안전운임 기준에 맞는지 자동 대사**한다.
- [ ] 거래명세서(사진/PDF/엑셀) 업로드 → AI가 각 줄 항목 추출
      (목적지·항구·컨테이너 규격·운송유형·할증·청구금액)
- [ ] 추출한 조건으로 **앱의 운임 계산 엔진이 '올바른 금액' 산출** → 청구금액과 비교
- [ ] 결과표: 일치 / 불일치(과다·과소 청구) · 차액 · 근거(고시·할증) 표시
- 안전성: **AI는 추출만, '정답 금액'은 고시 기반 계산기가 산출**, 불일치 건은 사람이 검토
  → AI가 금액을 판단하지 않으므로 정확성 유지 (관통 원칙 그대로)
- 강점: 앱의 **운임·할증·복화·COMBINE 계산 로직을 그대로 검증 기준**으로 재사용
- 주의: 명세서 양식이 협력사마다 다름(AI 추출로 흡수) · 목적지명→행정동 매칭 확인 필요
        · 할증/품목 등 계산에 필요한 정보가 명세서에 없으면 보완 입력
- 대상 가치: 청구 과오 자동 적발 · 정산 신뢰성↑ · 수기 대사 시간 대폭 절감
- 연계: Phase 3(문서 추출 기반) + 운임 계산 엔진 → 자연스러운 확장

## 🎯 북극성(최종) — Phase 4: 배차 자동화 & 연계 최적 배차  ·  난이도: 상
오더가 생성되면 **가용 차량/기사에 자동 배차**하고, 나아가 **연계 배차로 효율을 최적화**한다.

- [ ] **배차 자동화**: 오더 → 가용 차량·기사 자동 매칭/배정
- [ ] **연계 최적 배차(효율 극대화)**: AI가 오더들을 묶어 빈차 운행 최소·차주 수익 최대화
  - 복화운송 매칭(왕복 화물 연결) — 앱이 이미 아는 *복화 = 왕복운임 100% 추가*
  - COMBINE 운송 매칭(20FT 2개 묶음) — *조건 충족 시 180%*
  - 공차 운행 최소화 — *공차 10km↑ = 왕복 50%* / 포지셔닝 50%
  - 편도(의왕ICD) 회送 경로 최적 연결
- 핵심 강점: **앱이 이미 가진 운임·할증·복화·COMBINE 로직**을 그대로 최적화 엔진의 제약/목표로 사용
- 의존성: 차량·기사·실시간 위치 데이터 + 사내 배차/운영 시스템 연동
- 대상 가치: 공차↓·복화↑ → **비용 절감 + 차주 수익 증대**(세방 운영 효율의 본질)

## 진행 순서 (의존성 기준)
1. Phase 1 (음성입력) — ☑ 1차 구현(검증 중)
2. Phase 2 (부대조항 QA) — 소형 API 확보 후
3. Phase 3 (B/L→오더) — 사내 연동 창구 확인 후
4. Phase 3.5 (거래명세서 대사) — 문서 추출 + 운임 계산 엔진 결합 (사내 정산 데이터 연계 시 강력)
5. Phase 4 (배차 자동화·최적 배차) — Phase 3 연동 기반 위에서 본격화

> 메모: 본 문서는 AI 확장 논의의 기록용 백로그. 각 항목 착수 시 별도 설계 문서로 구체화한다.
